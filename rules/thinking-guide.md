# Graduated Thinking Guide

Thinking level mapping by task type.

## Thinking Levels

### `think` — General implementation judgment
- Single function implementation
- Simple component writing
- Type definition
- Test writing

### `think hard` — API design, state management
- API endpoint design
- State management structure (ComponentStore state, Zustand store)
- Module boundary definition
- Data flow design
- Complex type systems (generics, utility types)

### `think harder` — Migration, complex refactoring
- Library migration strategy
- Large-scale refactoring planning
- Maintaining backward compatibility during changes
- Complex business logic redesign

### `ultrathink` — Architecture decisions, technology selection
- Project architecture decisions
- Technology stack selection (frameworks, state management, testing)
- Tradeoff analysis
- Long-term scalability judgment
- Performance vs maintainability decision

## Usage
- Enter Plan Mode (Shift+Tab), then specify thinking level
- Example: "think hard about the state management structure"
- Example: "ultrathink about whether we should use TanStack Query or SWR"
