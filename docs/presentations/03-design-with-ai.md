# AI 디자인 — 슬롭 탈출기

## 핵심 메시지
AI가 만든 UI는 왜 다 비슷할까? 그리고 어떻게 벗어나는가.

---

## 문제: AI 슬롭 (AI Slop)

AI에게 "멋진 랜딩 페이지 만들어줘"라고 하면 99% 이렇게 나옴:

```
- 보라색/파란색 그라데이션 배경
- Inter 폰트 + 흰 배경 + 회색 카드
- 3열 feature 그리드 (아이콘 + 제목 + 설명)
- 모든 걸 가운데 정렬
- "Get Started" 보라색 버튼
- 의미 없는 둥근 모서리와 그림자
```

왜? 학습 데이터에 이 패턴이 가장 많기 때문.
AI는 "가장 흔한 것"을 생성한다. "좋은 것"을 생성하는 게 아니다.

---

## 해결: 디자인 스킬로 편향 교정

### 접근 1: 금지 규칙으로 슬롭 차단

```
Inter 폰트 → 금지. Geist, Satoshi, Cabinet Grotesk 사용
보라색 그라데이션 → 금지. Zinc/Slate 베이스 + 단일 액센트
3열 그리드 → 금지. 비대칭 레이아웃, 벤토 그리드
가운데 정렬 Hero → 금지. Split Screen, 좌측 정렬
```

"하지마" 목록이 있으면 AI는 대안을 찾는다.

### 접근 2: 구체적 수치로 디자인 제어

```
DESIGN_VARIANCE: 8    (1=완전 대칭 ~ 10=예술적 혼돈)
MOTION_INTENSITY: 6   (1=정적 ~ 10=시네마틱)
VISUAL_DENSITY: 4     (1=갤러리 ~ 10=조종석)
```

"멋지게"가 아니라 숫자로 지시하면 결과가 일관된다.

### 접근 3: 레이아웃 랜덤화

AI는 매번 같은 레이아웃을 생성하는 경향이 있음.
gpt-taste 스킬은 Python RNG로 레이아웃을 랜덤 선택해서
매번 다른 구조가 나오도록 강제.

## Before / After

### Before (일반 AI)
```
[가운데 정렬 Hero]
[3열 Feature 카드]
[Testimonial 슬라이더]
[CTA 섹션]
[Footer]
← 어디서 많이 본 그 레이아웃
```

### After (디자인 스킬 적용)
```
[비대칭 Split Hero + GSAP 스크롤 애니메이션]
[벤토 그리드 Feature (다양한 카드 크기)]
[풀스크린 이미지 + 텍스트 오버레이]
[인터랙티브 CTA + 마이크로 인터랙션]
[미니멀 Footer]
← 실제 디자인된 느낌
```

## 핵심 원리

1. **AI 슬롭은 학습 데이터 편향** — "좋은 디자인"이 아니라 "흔한 디자인"을 생성
2. **금지 규칙이 효과적** — "멋지게 해줘"보다 "보라색 쓰지마, Inter 쓰지마"가 낫다
3. **수치 제어** — 추상적 형용사 대신 구체적 숫자로 지시
4. **스킬은 편향 교정 도구** — AI의 디폴트를 프리미엄 쪽으로 강제 이동

## 한줄 요약
> AI에게 "멋지게"라고 하면 슬롭이 나온다. "이건 하지마, 이 수치로 해"라고 하면 디자인이 나온다.
